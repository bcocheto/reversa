# Context

O `context/` concentra o conhecimento durável do projeto.
Use esta pasta para manter explicações que ajudam humanos e agentes a entender o sistema sem precisar abrir todo o código.

## Inclua aqui

- Visão geral do produto.
- Arquitetura principal.
- Convenções e padrões.
- Estratégia de testes.
- Notas de implantação.
- Glossário dos termos mais usados.

## Atualização

- Reescreva quando o projeto mudar de direção.
- Prefira exemplos curtos e legíveis.

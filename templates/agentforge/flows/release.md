# Release

## Objetivo

Promover uma entrega segura e rastreavel.

## Quando usar

- O usuario pediu publicacao, deploy ou rollout.

## Etapas

1. Validar o estado atual.
2. Preparar o pacote de saida.
3. Publicar ou promover a entrega.
4. Revisar o resultado.

## Checkpoints

- Validação concluída.
- Exportação concluída.
- Publicação verificada.

## Conclusão

- A entrega foi promovida com segurança.

# Feature Development

## Objetivo

Entregar uma feature nova com clareza de escopo, arquitetura e revisão.

## Quando usar

- O usuário pediu uma funcionalidade nova.
- A mudança exige design, implementação e revisão sequenciais.

## Etapas

1. Descobrir o problema.
2. Refinar a solução.
3. Definir arquitetura.
4. Implementar.
5. Revisar.

## Checkpoints

- Escopo aprovado.
- Arquitetura aprovada.
- Implementação validada.
- Revisão concluída.

## Conclusão

- A funcionalidade está pronta para uso.
